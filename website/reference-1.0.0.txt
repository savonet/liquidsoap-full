h2. Liquidsoap scripting language reference

h3. Categories

The **Source / ...** categories contain all functions that return sources.
The **Input** functions are those which build elementary sources
(playing files, synthesizing sound, etc.).
The **Output** functions are those which take a source and register it
for being streamed to the outside (file, soundcard, audio server, etc.).
The **Visualization** functions are experimental ones that let you 
visualize in real-time some aspects of the audio stream.
The **Sound Processing** functions are those which basically work on the source 
as a continuous audio stream. They would typically be mixers of streams,
audio effects or analysis.
Finally, **Track Processing** functions are basically all 
others, often having a behaviour that depends on or affects the extra 
information that liquidsoap puts in streams: track limits and metadata.

h5@input_harbor. input.harbor
<pre>(?id:string,?auth:((string,string)->bool),?buffer:float,
 ?debug:bool,?dumpfile:string,?icy:bool,
 ?icy_metadata_charset:string,?logfile:string,?max:float,
 ?metadata_charset:string,
 ?on_connect:(([(string*string)])->unit),
 ?on_disconnect:(()->unit),?password:string,?port:int,
 ?timeout:float,?user:string,string)->
source(audio='#a+1,video=0,midi=0)</pre>

Retrieves the given http stream from the harbor.

* <code>id</code> (<code>string</code> -- defaults to <code>""</code>): Force the value of the source ID.
* <code>auth</code> (<code>(string,string)->bool</code> -- defaults to <code>fun (_,_) -> false</code>): Authentication function. <code>f(login,password)</code> returns <code>true</code> if the user should be granted access for this login. Override any other method if used.
* <code>buffer</code> (<code>float</code> -- defaults to <code>2.</code>): Duration of the pre-buffered data.
* <code>debug</code> (<code>bool</code> -- defaults to <code>false</code>): Run in debugging mode by not catching some exceptions.
* <code>dumpfile</code> (<code>string</code> -- defaults to <code>""</code>): Dump stream to file, for debugging purpose. Disabled if empty.
* <code>icy</code> (<code>bool</code> -- defaults to <code>false</code>): Enable ICY (shoutcast) protocol.
* <code>icy_metadata_charset</code> (<code>string</code> -- defaults to <code>""</code>): ICY (shoutcast) metadata charset. Guessed if empty. Default for shoutcast is ISO-8859-1. Set to that value if all your clients send metadata using this charset and automatic detection is not working for you.
* <code>logfile</code> (<code>string</code> -- defaults to <code>""</code>): Log buffer status to file, for debugging purpose. Disabled if empty.
* <code>max</code> (<code>float</code> -- defaults to <code>10.</code>): Maximum duration of the buffered data.
* <code>metadata_charset</code> (<code>string</code> -- defaults to <code>""</code>): Metadata charset for non-ICY (shoutcast) source protocols. Guessed if empty.
* <code>on_connect</code> (<code>([(string*string)])->unit</code> -- defaults to <code>fun (_) -> ()</code>): Function to execute when a source is connected. Its receives the list of headers, of the form: (<label>,<value>). All labels are lowercase.
* <code>on_disconnect</code> (<code>()->unit</code> -- defaults to <code>{()}</code>): Functions to excecute when a source is disconnected
* <code>password</code> (<code>string</code> -- defaults to <code>"hackme"</code>): Source password.
* <code>port</code> (<code>int</code> -- defaults to <code>8005</code>): Port used to connect to the source.
* <code>timeout</code> (<code>float</code> -- defaults to <code>30.</code>): Timeout for source connectionn.
* <code>user</code> (<code>string</code> -- defaults to <code>"source"</code>): Source user.
* <code>(unlabeled)</code> (<code>string</code>): Mountpoint to look for.

h5@input_http. input.http
<pre>(?id:string,?autostart:bool,?bind_address:string,
 ?buffer:float,?debug:bool,?force_mime:string,
 ?logfile:string,?max:float,?new_track_on_metadata:bool,
 ?playlist_mode:string,?poll_delay:float,?timeout:float,
 ?user_agent:string,string)->
source(audio='#a,video='#b,midi=0)</pre>

Forwards the given http stream. The relay can be paused/resumed using the start/stop telnet commands.

* <code>id</code> (<code>string</code> -- defaults to <code>""</code>): Force the value of the source ID.
* <code>autostart</code> (<code>bool</code> -- defaults to <code>true</code>): Initially start relaying or not.
* <code>bind_address</code> (<code>string</code> -- defaults to <code>""</code>): Address to bind on the local machine. This option can be useful if your machine is bound to multiple IPs. Empty means no bind address.
* <code>buffer</code> (<code>float</code> -- defaults to <code>2.</code>): Duration of the pre-buffered data.
* <code>debug</code> (<code>bool</code> -- defaults to <code>false</code>): Run in debugging mode, not catching some exceptions.
* <code>force_mime</code> (<code>string</code> -- defaults to <code>""</code>): Force mime data type. Not used if empty.
* <code>logfile</code> (<code>string</code> -- defaults to <code>""</code>): Log buffer status to file, for debugging purpose. Disabled if empty.
* <code>max</code> (<code>float</code> -- defaults to <code>10.</code>): Maximum duration of the buffered data.
* <code>new_track_on_metadata</code> (<code>bool</code> -- defaults to <code>true</code>): Treat new metadata as new track.
* <code>playlist_mode</code> (<code>string</code> -- defaults to <code>"normal"</code>): Valid modes are "normal", "random", "randomize" and "first". The first ones have the same meaning as for the mode parameter of the playlist operator. The last one discards all entries but the first one.
* <code>poll_delay</code> (<code>float</code> -- defaults to <code>2.</code>): Polling delay when trying to connect to the stream.
* <code>timeout</code> (<code>float</code> -- defaults to <code>30.</code>): Timeout for source connectionn.
* <code>user_agent</code> (<code>string</code> -- defaults to <code>"Liquidsoap/1.0.0 (Unix; OCaml 3.12.0)"</code>): User agent.
* <code>(unlabeled)</code> (<code>string</code>): URL of an http stream (default port is 80).

* <code>client</code> (<code>string</code> -- defaults to <code>"liquidsoap"</code>)
* <code>clock_safe</code> (<code>bool</code> -- defaults to <code>true</code>): Force the use of the dedicated Pulseaudio clock.
* <code>device</code> (<code>string</code> -- defaults to <code>""</code>): Device to use. Uses default if set to "".
* <code>fallible</code> (<code>bool</code> -- defaults to <code>false</code>): Allow the input to stop. When false, the source will be infallible but the stop command won't have any effect.
* <code>fallible</code> (<code>bool</code> -- defaults to <code>false</code>): Allow the child source to fail, in which case the output will be (temporarily) stopped.
* <code>on_start</code> (<code>()->unit</code> -- defaults to <code>{()}</code>): Callback executed when input starts.
* <code>on_start</code> (<code>()->unit</code> -- defaults to <code>{()}</code>): Callback executed when outputting starts.
* <code>on_stop</code> (<code>()->unit</code> -- defaults to <code>{()}</code>): Callback executed when input stops.
* <code>on_stop</code> (<code>()->unit</code> -- defaults to <code>{()}</code>): Callback executed when outputting stops.
* <code>start</code> (<code>bool</code> -- defaults to <code>true</code>): Start input as soon as it is created. Disabling it is only taken into account for a fallible input.
* <code>start</code> (<code>bool</code> -- defaults to <code>true</code>): Automatically start outputting whenever possible. If true, an infallible (normal) output will start outputting as soon as it is created, and a fallible output will (re)start as soon as its source becomes available for streaming.

h5@cross. cross
<pre>(?id:string,?active:bool,?conservative:bool,
 ?duration:float,?inhibit:float,?minimum:float,
 ?override:string,((source('a),source('a))->source('a)),
 source('a))->source('a)</pre>

Generic cross operator, allowing the composition of the N last seconds of a track with the beginning of the next track.

* <code>id</code> (<code>string</code> -- defaults to <code>""</code>): Force the value of the source ID.
* <code>active</code> (<code>bool</code> -- defaults to <code>false</code>): The active behavior is to keep ticking the child's clock when the operator is not streaming. Otherwise the child's clock is strictly based on what is streamed off the child source, which results in time-dependent active sources to be frozen when that source is stopped.
* <code>conservative</code> (<code>bool</code> -- defaults to <code>false</code>): Do not trust remaining time estimations, always buffering data in advance. This avoids being tricked by skips, either manual or caused by skip_blank().
* <code>duration</code> (<code>float</code> -- defaults to <code>5.</code>): Duration in seconds of the crossed end of track. This value can be changed on a per-file basis using a special metadata field.
* <code>inhibit</code> (<code>float</code> -- defaults to <code>-1.</code>): Minimum delay between two transitions. It is useful in order to avoid that a transition is triggered on top of another when an end-of-track occurs in the first one. Negative values mean <code>duration+1</code>. Warning: zero inhibition can cause infinite loops.
* <code>minimum</code> (<code>float</code> -- defaults to <code>-1.</code>): Minimum duration (in sec.) for a cross: If the track ends without any warning (e.g. in case of skip) there may not be enough data for a decent composition. Set to 0. to avoid having transitions after skips, or more to avoid transitions on short tracks. With the negative default, transitions always occur.
* <code>override</code> (<code>string</code> -- defaults to <code>"liq_start_next"</code>): Metadata field which, if present and containing a float, overrides the 'duration' parameter for current track.
* <code>(unlabeled)</code> (<code>(source('a),source('a))->source('a)</code>): Composition of an end of track and the next track.
* <code>(unlabeled)</code> (<code>source('a)</code>)
